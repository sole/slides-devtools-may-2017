devtools/

It's like a website

the client (what you see) connects to the server and asks for data

except for the fact that websites have APIs, but we have a protocol

The Remote Debugging Protocol

RDP

					
					firefox -P development -console

(launches Firefox and opens a console showing all the logs)

Who is the server?

Anything that can be debugged:

  • Firefox Desktop
  • Firefox for Android
  • (you might remember) Firefox OS devices
  • (with adapters) other things/anything you implement

Things we ask to the server

  • What can we inspect? (capabilities)
  • List of tabs
  • Data about a tab
  • Etc...

Debugging servers or connections aren't necessarily fast, so it's better to ask for data and render/process it in the client

Things that run on the client

  • The DevTools UI
  • Rendering graphs about debugging data
  • Rendering HTML trees, filtering data, etc
  • The debugger!
  • The console!
  • etc etc...
